Algorithm 1 Supervised Non-linear Centroid-Encoder (without pre-training)

  1. Dashboard
  2. Bootstrap Grid
Be sure to visit Bootstrap's grid documentation for more information.

To begin this algorithm of Supervised Non-linear Centroid without pre-training, we will use the labeled data (D= {xi}) that consist of our input vectors {xij} with M classes.

Input: In our input for the algorithm, notice the set of parameters that were defined. The (τ) parameter is considered the error tolerance which is used to determine when the algorithm should stop iterating. The (μ) parameter represents the learning rate, which is used to update the states. The bottleneck dimension m represents the dimensionality of our output vector yi from the bottleneck layer of the network.

Output: For the Bottleneck Output, yi, represents the output from the network function g(xi) which transforms the input vector xi into a new form. The Network Parameter (θ) represents all of unknown parameters the weights and biases that are being optimized in the network.

Result: From the output, the results show the non-linear embedding of data in m dimensions. This is shown by the algorithm mapping the input data into a new m- dimensional space that causes thes the classes to be more separable and represent the underlying patterns.

Initialization Pre-Steps: Class centroids are the points in a featured space that reprsents the average position of all the data points belonging to a particular class j in which it acts as a representative center for the class. To calculate the class centroids, first set Cj to represent all the data points that belongs to class j. The centroid of each class is defined as |Cj| is the cardinality of class Cj which is the number of elements in class j. According to "UML Class Diagrams", it states," Cardinality concept indicates the number of instances of one class linked to the instances of another class. This can be simply expressed as a one to one, one to many or many to many relationship." Establishing the index set Ij, we will use this to set the indices to refer to all the data points that are members of class j.

Initializing Steps: Begin with the splitting of the data into a training set Tr and validation set V.Start the iteration process at t=0 so that for each iteration, we can check if the change in validation loss |LCt+1(V)-LCt(C)| is greater than (τ). If so, compute the loss/error for LCV and LCTr in Equation 1:

LC(D;(θ))= 1/(2N) ∑ j=1M ∑ i ε Ij ||cj-f(xi; θ) ||22

where f is the mapping with the ordered input-output pairs. Then compute the backpropagation error (chain rule) Δ to get the gradient (derivative) of th loss w.r.t. to the network parameters θ C using Equation 3: Highlighted Code

ΔC=1/N ∑ j=1 (∑ i ε Ij f(xi;θ)-cj)

on the training set Tr.

Following the computation of the backpropagation, we can update the model parameters θ using the learning rate μ and gradient ΔC using the updating rule θt+1=θt-μΔC*gt. If the conditions are not met in the first step (|LCt+1(V)-LCt(C)| is greater than (τ)), the iterations stop.
