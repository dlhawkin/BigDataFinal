Methods in Class Sparse PCA

  1. Dashboard

Sparse Principal Component Analysis (sparse PCA) and Centroid Encoders both extend the traditional PCA to address challenges in data analysis. Sparse PCA enhances the interpretability of PCA by incorporating L1 regularization, which promotes sparsity in the loading vectors (Zou, Hastie, & Tibshirani, 2006). This adaptation ensures that each principal component relies only on a limited subset of features so that the results are easier to interpret, which is especially useful in bioinformatics contexts (Shen & Huang, 2008). On the other hand, Centroid-Encoders modify the autoencoder framework to focus on minimizing the distance between the network's output and the predefined class centroids, instead of reconstructing the input (McConville, Santos-Rodriguez, Piechocki, & Craddock, 2019). This approach utilizes a supervised learning method where the class labels directly guide the training process. The main application of CE is in supervised dimensionality reduction and data visualization, where maintaining class separability in the reduced-dimensional space is vital (LeCun, Bengio, & Hinton, 2015), such as in customer segmentation or medical imaging data (Litjens et al., 2017).

While both methods aim to improve the usability of PCA, they do so from different angles. Sparse PCA is unsupervised and enhances feature interpretability by creating sparse components (Zou, Hastie, & Tibshirani, 2006). Conversely, Centroid-Encoders use class labels to ensure that the reduced-dimensional space distinctly represents different classes. This makes CE a supervised technique,and useful for scenarios where labels are informative for the analysis. Both techniques provide more meaningful data representations by building on the foundational principles of PCA, tailored to meet specific analytical needs.
