Introduction

A Centroid-Encoder (CE) is a refined version of traditional neural networks. It enhances dimensionality reduction by incorporating class labels into its architecture. Unlike standard neural networks that capture broad data structures, CE focuses on emphasizing class-specific features. This goal is achieved by directing the network’s learning to minimize distances between data points and their class centroids in a reduced dimensional space.

Traditional autoencoders excel at compressing data into a manageable form. However, they often overlook external labels. They aim to reconstruct input data at their output, preserving information across a bottleneck layer. This layer is central to the processes of data compression and expansion. In contrast, the Centroid-Encoder introduces a supervised element by utilizing class labels to guide the compression process. It associates each class label with a centroid—a central point the CE strives to approximate within the compressed space. This strategy preserves data integrity. It also enhances the model's utility for predictive tasks by improving class separability. This is crucial in applications for things such as medical imaging diagnosis.

Centroid-Encoders organize the encoded representations around class centroids. This leads to a more structured and interpretable reduced space, where similar items cluster more distinctly based on their categories. This organization boosts the performance of subsequent classification algorithms, making post-processing analysis easier and more effective.

Objective: The primary goal is to optimize the model's parameters θ so that the outputs (fxi; θ), representing the predicted positions for each data point xi, closely align with their respective class centroids cj. By doing this, the loss function (LC(D; θ) is minimized, and the model accurately clusters the data points into their true classes with minimized distance from their centroids.

Here’s what each part of this equation means:

(LC(D; θ): This is the cost function for the centroid-encoder. It measures the overall error/loss that the model needs to minimize during the training process.

(D): Represents the dataset containing all data points xi.

(θ): These are the parameters, like weights and biases, that the model adjusts to reduce the cost.

(N): The total number of data points in the dataset.

(M): The number of different classes.

(cj): The centroid of class (j). This is the target where all data points in class (j) should ideally cluster.

(f(xi; θ)): This is the model's output for data point xi after being processed by the function (f) with parameters θ.

Summation Symbol

(∑j=1M and ∑ ε Ij): These summations calculate the error for each data point in each class and then add these errors up across all classes. The first summation ∑j=1M is over classes, ensuring we capture the error for every class. The second summation ∑i ε Ij is within a class, summing up the errors for each data point in that class.

(|cj - f(x; θ)|): This calculates the squared Euclidean distance between the model’s prediction for data point (xi) and the centroid (cj) of its class. Squaring the distance ensures that larger errors have a disproportionately higher impact on the total cost.
